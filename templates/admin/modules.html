Modules registered with screen-mgr. Service modules can be started/stopped from here.
Add external module
Point at a JSON manifest URL hosted by another service. It will be fetched, validated, and registered.
What is an external module?

A small service running elsewhere on the LAN (a Raspberry Pi, a colleague's laptop, a mini PC, whatever) that wants to plug into screen-mgr — for example, a robot control panel, a sensor dashboard, or another LED display. The service hosts a single JSON file (the manifest) that tells screen-mgr how to talk to it. You paste that JSON's URL below, click Register, and the module appears in this list. screen-mgr never runs the external module's code — it only fetches the manifest and calls the URLs the manifest declares.

How to do it
  1. Your service hosts a JSON file at some URL (e.g. http://robotpi.local:8080/manifest.json).
  2. Paste that URL into the input below.
  3. Click Register. The manifest is fetched, validated, and the module appears in the list.
  4. The new module's status updates from its health_url on each refresh tick.
  5. Remove it with the ✕ button next to the module row.
Display-module manifest

For a module that supplies content for a screen to show:

{
  "id": "robot-panel",
  "name": "Robot Control Panel",
  "description": "Live status + drive control for the lab robot",
  "type": ["display"],
  "version": "0.2.0",
  "health_url": "http://robotpi.local:8080/health",
  "screen_url_pattern": "http://robotpi.local:8080/screen/{screen_id}"
}
Service-module manifest

For a backend service the admin can start/stop (no screen content):

{
  "id": "co2-fan",
  "name": "Cleanroom CO2 Fan",
  "description": "Toggles the ceiling fan via a relay",
  "type": ["service"],
  "version": "0.1.0",
  "health_url": "http://co2pi.local:8090/health",
  "start_url": "http://co2pi.local:8090/start",
  "stop_url": "http://co2pi.local:8090/stop"
}
Manifest field reference
Field Required Notes
idyesStable slug. Must be unique. For display modules this is also the Screen.type value.
nameyesHuman-friendly label shown in admin.
descriptionnoOne-liner shown under the name.
typeyesArray containing "display" and/or "service". First match wins in v1.
versionnoSemver-ish string. Shown in the admin row.
health_urlrecommendedGET on each availability tick; 2xx response → green badge. Omit and the module is always "available".
screen_url_patterndisplay onlyURL template a screen loads. {screen_id} and {screen_name} placeholders are substituted at render time.
start_urlservice onlyscreen-mgr POSTs here when Start is clicked.
stop_urlservice onlyscreen-mgr POSTs here when Stop is clicked.
