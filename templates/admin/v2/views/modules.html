Modules

Every controllable thing in the system — built-in primitives, services, and externally-registered manifests. Each row is one module.

{# ---------- Add external module card ---------- #}

Add external module

Point at a JSON manifest URL hosted by another service. It will be fetched, validated, and registered.
{# Help panel #}

What is an external module? A small service running elsewhere on the LAN that wants to plug into the Studio (a robot dashboard, sensor readout, etc.). screen-mgr fetches its manifest, never its code.

Display-module manifest example:

{
  "id": "robot-panel",
  "name": "Robot Control Panel",
  "type": ["display"],
  "version": "0.2.0",
  "health_url": "http://robotpi.local:8080/health",
  "screen_url_pattern": "http://robotpi.local:8080/screen/{screen_id}"
}

Service-module manifest example:

{
  "id": "co2-fan",
  "name": "Cleanroom CO2 Fan",
  "type": ["service"],
  "version": "0.1.0",
  "health_url": "http://co2pi.local:8090/health",
  "start_url": "http://co2pi.local:8090/start",
  "stop_url": "http://co2pi.local:8090/stop"
}
